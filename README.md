# 🐴 馬開腹手術 判断支援アプリ（MVP）

結腸捻転・小腸絞扼の術中に、腸管の「切除すべきか否か」の判断を **支援・記録** する
Streamlit アプリです。

> ⚠️ **本アプリは診断・切除判断を行う装置ではありません。**
> 漿膜色の視覚評価は最も簡便だが最も不正確とされ、整復後に色が回復しても粘膜生存性とは
> 相関しません。本アプリは外科医の判断材料を客観的に数値化し、所見を整理・記録するための
> **研究・記録目的の試用ツール**です。最終的な切除可否は必ず術者が判断してください。

---

## できること（1症例＝1記録に統合）

1画面で上から順に入力し、1ボタンでまとめて保存します。

| ステップ | 中身 |
|------|------|
| 1. 撮影と色診断 | 結腸捻転=①結腸全体（漿膜）②結腸骨盤曲・切開部の粘膜 の2か所／小腸絞扼=切除ライン1か所。各撮影でROIの RGB・HSV・赤み/暗さ指標と「生存寄り/判定不能/壊死寄り」3段階ヒントを表示 |
| 2. 検査値 | Ht・血中乳酸・白血球数・腹水中乳酸・血中K を**実数値で入力**（未測定は空欄可） |
| 3. 最終判断・転帰 | 術者の判断、転帰、メモ |
| 4. 保存 | 上記すべてを1症例として `data/` に保存（1症例1ファイル＝後で研究データとして集計しやすい） |

年60頭・300症例が貯まれば研究データとして価値が出る設計。機械学習も OpenCV も使っていません（**Pillow + NumPy** のみ）。

## セットアップ

```powershell
# 1. このフォルダへ移動
cd C:\Users\d-miy\equine-surgery-app

# 2. （任意）依存を確認・インストール。多くは導入済み
python -m pip install -r requirements.txt

# 3. 起動
streamlit run app.py
```

ブラウザで `http://localhost:8501` が開きます。

## スマホ（iPhone / Android）で使う — HTTPS が必須

iPhone Safari は **HTTPS でないとカメラを起動しません**。実機テストには以下のいずれか。

### 方法A: 同一Wi-Fi内でPCにアクセス（カメラ不要の動作確認向け）
1. PCで `streamlit run app.py` 実行。起動時に表示される **Network URL**（例 `http://192.168.x.x:8501`）をスマホのブラウザで開く。
2. ※ これは http なのでiPhoneではカメラ不可。色解析は「ファイルから選択」で写真アップロードして試せます。

### 方法B: ngrok で HTTPS 化（カメラを使う実機テスト向け・推奨）
1. [ngrok](https://ngrok.com/) を入手しサインイン（無料枠で可）。
2. アプリ起動後、別ターミナルで:
   ```powershell
   ngrok http 8501
   ```
3. 表示される `https://xxxx.ngrok-free.app` をスマホで開く → カメラが使えます。

### ホーム画面に追加（アプリのように起動）
スマホのブラウザでページを開き「ホーム画面に追加」すると、アイコンから起動できます（PWA的な使い方）。

## 手術室での使い方のコツ
- 術者は滅菌グローブで画面を触れません。**助手が操作する**前提で。操作は「撮影 → 結果確認」の最小限に。
- 結腸捻転: 結腸骨盤曲の漿膜と粘膜を撮影。整復前に「基準として保存」→ 整復後に赤み回復率を確認。
- 小腸絞扼: 切除ライン（健常部と病変部の境界）を撮影。チェックリストで「境界の明瞭さ」「腸間膜動脈拍動」を重視。

---

## データの取り扱い（重要）
- 馬主・個人を特定できる情報は入力・保存しないでください。症例IDは匿名（日時＋乱数）。
- データは PC 内の `data/` にのみ保存され、外部に送信されません。`.gitignore` で除外済み。
- 院内での試用前に「研究・記録目的、正式な診断機器ではない」ことを関係者に共有してください。
