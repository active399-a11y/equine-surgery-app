# analysis/ — 研究用 集計スターター

馬の開腹手術（結腸捻転・小腸絞扼）判断支援アプリが出力する症例CSVを読み込み、
将来 300 症例貯まったときに「**切除した群 vs 温存した群**」で色値・検査値を
比較するための、最小限の集計スクリプトです。

> このディレクトリは後解析専用です。アプリ本体（`surgery_app/`, `app.py`）は変更しません。

## 使い方

```bash
# 引数にCSVのパスを渡す
python analysis/summarize_cases.py path/to/equine_cases.csv

# 省略すると ./equine_cases.csv を読む
python analysis/summarize_cases.py
```

実行すると標準出力に要約が表示され、群別要約のまとめが
`analysis/summary_output.csv`（UTF-8 BOM付き、Excelでそのまま開ける）に保存されます。

## 入力CSVについて

アプリの「症例記録」からエクスポートされるCSVを想定しています。
**1行 = 1症例**、列は日本語名で、`surgery_app/config.py` の `columns()` が定義します。

主な列:

| 区分 | 列名（例） |
| --- | --- |
| メタ | `症例ID` / `保存日時` / `病態` / `最終判断` / `転帰` |
| 検査値 | `Ht（ヘマトクリット）(%)` / `血中乳酸値(mmol/L)` / `白血球数(/µL)` / `腹水中乳酸値(mmol/L)` / `血中カリウム（K）(mEq/L)` |
| 色指標 | `① 結腸全体（漿膜）_赤み` / `..._暗さ` / `..._色相H` / `..._彩度S` / `..._明度V` / `..._色ヒント` ほか撮影部位ごと |
| その他 | `メモ` |

- **赤み** = R −(G+B)/2。高いほど赤い（ピンク/赤の強さ）。
- **暗さ** = 255 − 明度V換算。高いほど暗い（壊死域は暗くなる傾向）。

スクリプトは `surgery_app.config` を import できればその列名定義を使い、
できない場合も上記の文字列でフォールバック動作します。**列が欠けていても
クラッシュせず、存在する列だけ集計します。**

## 出力（標準出力）

1. 総症例数 / 病態別・転帰別・最終判断別の件数。
2. **最終判断「切除する」= 切除群、それ以外 = 非切除群** に二分し、検査値5項目の
   `n / mean / median` を群別比較。
3. 同じ二分で、色指標（存在する `_暗さ` と `_赤み` 列すべて）の `mean/median` を群別比較。
4. 転帰別（**生存・経過良好** vs **死亡・安楽殺**）の検査値要約。

## 出力（`summary_output.csv`）

群別要約を縦に積んだまとめ。列はおよそ次の通り:

`区分, 指標, 切除群_n, 切除群_mean, 切除群_median, 非切除群_n, 非切除群_mean, 非切除群_median`
（転帰別の行は群名が `生存群` / `死亡・安楽殺群` になります）

`区分` 列で「検査値(最終判断別)」「色指標(最終判断別)」「検査値(転帰別)」を見分けられます。

## 300症例貯まったときの分析例

このスターターを土台に、次のような分析へ広げられます。

- **切除群 vs 非切除群の検査値差**: 血中乳酸・腹水中乳酸の平均/中央値に差があるか。
  腹水中乳酸 − 血中乳酸の差分（腸管局所の虚血指標）を新列として比較。
- **色指標と判断の関係**: 切除群の `_暗さ` が非切除群より高い／`_赤み` が低いか。
  閾値（`color_analysis.viability_hint` の仮値）の妥当性検証に使う。
- **転帰との突き合わせ**: 「切除した／温存した」判断が転帰（生存/死亡・安楽殺）に
  どう繋がったか。色値・検査値のどれが転帰をよく分けるか（ROC/箱ひげ図など）。
- **症例報告・将来のAI学習データ**: ここで整えた群別の表が、症例報告の素材や
  機械学習データセットの第一歩になります。

> 注意: 本アプリは診断機器ではなく、客観的な記録・支援ツールです。
> 集計結果も「最終判断は術者」という前提で解釈してください。
